# Fast and Reliable Small-Data Learning: LLM-Driven Sign-Constrained Kernel Machines

This repository contains the official implementation of the paper:
"Fast and Reliable Small-Data Learning: LLM-Driven Sign-Constrained Kernel Machines".

## Prerequisites

This package depends on Gemini, Matlab, and Python. Operations have been verified in the following environment:
- OS:
  - Verified: Ubuntu 22.04
- Matlab: R2025b
  - Required licenses: Matlab Base, Optimization Toolbox, Statistics and Machine Learning Toolbox
- Python: 3.10 or higher
  - Package management: `uv` (the officially supported procedure for this repository uses `uv` only)
  - C compiler: Required for Cython build (`gcc`, etc. On Ubuntu, use the `build-essential` package)
- Node.js: `>= 18`
  - Required for running Gemini CLI

## Setup for the llm and python Folders

The `llm` folder is used to obtain the results for Table 3.
The `python` folder is used to obtain the results for Figure 4.
For both the `llm` and `python` folders, the following preparation is required first.

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```
Then, navigate to the `llm` or `python` folder respectively and install the dependency packages:
```
uv sync --frozen
```

The following additional preparation is also required under the `llm` folder.

Install Gemini CLI:
```bash
npm install -g @google/gemini-cli
```
One of the following authentication methods is required to use Gemini CLI:

1. API key authentication (recommended): Set the environment variable `GEMINI_API_KEY`.

   ```bash
   export GEMINI_API_KEY="your-api-key-here"
   ```

2. Google login authentication: Perform browser authentication with `gemini auth login`.

Setting the environment variable `GEMINI_API_KEY` is also required.

For Step 4 (`run_step4.py`), API keys are required via environment variables:

```bash
export OPENAI_API_KEY="your-openai-api-key"
export ANTHROPIC_API_KEY="your-anthropic-api-key"
export GEMINI_API_KEY="your-gemini-api-key"
```

If running Step 4 with the default `--provider all`, all three keys are required.

The following additional preparation is also required under the `python/runtime` folder.
```bash
# Install the proxqp solver
uv pip install proxsuite

# Build the Cython module (only if necessary)
# Not required if an environment-compatible pfgd_solvers*.so already exists
cd src && uv run python setup.py build_ext --inplace && cd ..
```

> **Note**: Building is not necessary if the import of `pfgd_solvers` succeeds. Build only if you encounter `ImportError: Cython module 'pfgd_solvers' is not available`.
> A rebuild is also required if the Python major/minor version or ABI does not match the `.so` file.

## Tables 1, 3 (Sign constraints)

Automatic generation of sign constraints consists of the following 5 steps. Each step is executed on a per-dataset basis.

### Pipeline Overview

```
Step 1 (Gemini)  →  Step 2 (Gemini)  →  Step 3 (Offline)  →  Step 4 (LLM API)  →  Step 5 (Offline)
  Abstraction of       Data                Input JSON           Sign constraint      .mat export
  dataset overview     preprocessing       generation           inference
```

### Step 1: Abstraction of Dataset Overview (Gemini CLI required)

Generates a generalized description with proper nouns removed from the dataset overview text, in order to eliminate LLM prior knowledge bias.

```bash
uv run scripts/run_step1.py "adult"
```

- Input: `dataset_assets/dataset_overview/adult_overview.txt`
- Output: `Generalized_dataset_info/adult_info.txt`

### Step 2: Data Preprocessing (Gemini CLI required)

Has the LLM automatically generate and execute a preprocessing script. This includes handling missing values, encoding categorical variables, binarizing the target variable, and other operations.

```bash
uv run scripts/run_step2.py "adult"
```

- Input: `dataset_assets/raw_data/adult_raw.csv`, output from Step 1
- Output: The following files are generated under `preprocessed_dat/adult/`
  - `preprocessed.csv` — Preprocessed data
  - `preprocessing_report.txt` — Preprocessing report
  - `schema.json` — Schema information
  - `preprocessing_script.py` — Preprocessing script generated by the LLM

### Step 3: Input JSON Generation (Offline, Gemini not required)

Assembles a JSON file as input for Step 4 from the preprocessed data and the abstracted description.

```bash
uv run scripts/run_step3.py "adult"
```

- Input: `Generalized_dataset_info/adult_info.txt`, `preprocessed_dat/adult/`
- Output: `input_json/adult_input.json`

### Step 4: Automatic Inference of Sign Constraints (LLM API required)

The LLM infers sign constraints (`-1` / `0` / `+1`) and their rationale for each feature based on domain knowledge, and outputs provider/model-specific JSON files.

```bash
uv run scripts/run_step4.py "adult"
```

Example (single provider/model):

```bash
uv run scripts/run_step4.py "adult" --provider openai --model gpt-5.4 --overwrite
```

- Main options:
  - `--provider {openai,anthropic,gemini,all}` (default: `all`)
  - `--model MODEL_NAME` (for single-provider runs)
  - `--temperature` (default: `0.0`), `--max-retries` (default: `2`), `--timeout` (default: `600`)
  - `--max-output-tokens` (default: `65536`), `--overwrite`, `--dry-run`, `--log-dir`

- Input: `input_json/adult_input.json`
- Output:
  - `sc_json/openai/gpt-5.4/adult_sc.json`
  - `sc_json/anthropic/claude-sonnet-4-6/adult_sc.json`
  - `sc_json/gemini/gemini-3.1-pro-preview/adult_sc.json`

Step 4 logs are written under `logs/step4_api/` (`raw/`, `normalized/`, `requests.jsonl`, `summary.csv`).

Tables 1 and 3 are transcribed from these results.

### Step 5: MATLAB Format Export (Offline, Gemini not required)

Combines the preprocessed data and sign constraints, and outputs provider/model-specific `.mat` files for downstream analysis.

```bash
uv run scripts/run_step5.py "adult"
```

- Input: `preprocessed_dat/adult/preprocessed.csv`, `sc_json/*/*/adult_sc.json`
- Output:
  - `mat_data/openai/gpt-5.4/uci_adult.mat`
  - `mat_data/anthropic/claude-sonnet-4-6/uci_adult.mat`
  - `mat_data/gemini/gemini-3.1-pro-preview/uci_adult.mat` (if corresponding JSON exists)
  - Contained variables: `X_dat` (feature matrix), `y_dat` (target), `cvec_sgn` (sign constraint vector)

The experimental results in the paper were obtained by taking 10 times the Z-score of the matrix `X_dat` obtained here.

## Tables 2, 5, 6 (Accuracy comparison with monotone GBDT)

Navigate to the `python/compare_accuracy` folder and install dependencies:

```bash
cd python/compare_accuracy
uv sync --frozen
```

Run the comparison script:

```bash
uv run script/compare_acc.py
```

This script runs 200 random CV splits for each dataset at `n in {5, 10}` and compares:

- Sign-constrained / sign-free kernel methods (`sc-lin`, `sc-rbf`, `sf-lin`, `sf-rbf`)
- Monotonic-constrained / monotonic-free LightGBM baselines (`mc-lightgbm`, `mf-lightgbm`)

Input data are read from `python/compare_accuracy/data/*.mat`.

After execution, outputs are generated under `python/compare_accuracy/out-demo1321-02/<timestamp>/`:

| File                     | Format | Content                                                                 |
| ------------------------ | ------ | ----------------------------------------------------------------------- |
| `percv_accs.npz`         | NumPy  | Raw per-CV accuracy arrays for all methods / datasets / training sizes |
| `results_<timestamp>.xlsx` | Excel  | Aggregated results and p-values (`Results`, `p-values (Wilcoxon)`, `p-values (t-test)`) |
| `compare_acc.py`         | Python | Snapshot of the executed script for reproducibility                     |

Tables 2, 5, and 6 in the paper are transcribed from `results_<timestamp>.xlsx`.

| `YYMMDDhhmmss_runtime.csv`  | CSV    | Average execution time by solver and number of constraints                 |

## Tables 7, 8 (Ablation study across LLM-derived constraints)

Navigate to the `python/ablation_study` folder and install dependencies:

```bash
cd python/ablation_study
uv sync --frozen
```

Run the ablation study script:

```bash
uv run script/run_ablation_study.py
```

Input data are read from `python/ablation_study/data/`.
The script expects provider-wise `.mat` files for each dataset under:

- `data/gemini/<model_name>/...*.mat`
- `data/openai/<model_name>/...*.mat`
- `data/anthropic/<model_name>/...*.mat`

For each dataset, the three provider files (Gemini/OpenAI/Anthropic) must all exist.

After execution, the following files are generated under `python/ablation_study/out-ablation-study/<timestamp>/`:

| File                        | Format | Content                                                                 |
| --------------------------- | ------ | ----------------------------------------------------------------------- |
| `result_ablation.xlsx`      | Excel  | Main ablation results (accuracy and p-value sheets for each training size) |
| `percv_accs_ablation.npz`   | NumPy  | Raw per-CV accuracy arrays for all kernels/methods/datasets             |
| `ablation_metadata.json`    | JSON   | Dataset-level metadata used during the run                              |
| `run_ablation_study.py`     | Python | Snapshot of the executed script for reproducibility                      |

Tables 7 and 8 in the paper are transcribed from `result_ablation.xlsx`
(corresponding to the default settings `--ntras 5 10`, i.e., two training-size conditions).
